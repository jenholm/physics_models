# Configuration-Space Locality Note

## What ISR uses

Configuration-space locality is used as a **constraint/prior on measure construction**.

Sectors receive weight based on:
- Shared local ancestry
- Field proximity
- Causal adjacency
- Common-origin structure with our observable patch

## What ISR does **not** claim

- This is **not** a claim about Bell-style local hidden variables.
- Configuration-space locality is a modeling prior, not an assertion about hidden variables.

## Safe language for the manuscript

> ISR uses locality as a weighting prior over common-origin inflationary sectors. It does not assert Bell-style local hidden variables.

## The diagnostic test

For many pairs $(x_i, x_j)$:

- **If** initial condition distance is small **and** same basin:
  - Then effective physics distance $d_\theta$ should usually be small.

- **If** initial condition distance is small **and** different basin:
  - Then $d_\theta$ may be large. This is expected.

- **If** initial condition distance is large:
  - Then $d_\theta$ is unconstrained.

## Failure condition

Small changes around $x_0$ produce chaotic $\theta$ without a basin explanation. That would falsify ISR's locality assumption.
